# 토스페이먼츠 테스트 결제 실패 케이스 정리

이 문서는 `test_` 키를 사용하는 테스트 환경에서 의도적으로 결제 실패를 유도하는 방법을 정리합니다.
정확한 실패 규칙은 토스 공식 문서를 기준으로 업데이트하세요.

## 빠른 실패 시나리오

- 결제창에서 `취소` 버튼 클릭 → 실패 콜백으로 이동
- 카드 유효기간을 과거로 입력 (예: `01/20`)

## 입력 검증 단계에서 실패시키기

- 카드번호 길이를 일부러 부족하게 입력
- 카드번호 체크섬이 맞지 않게 입력

## 승인 단계에서 실패시키기(환경에 따라 다름)

- CVC를 `000`으로 입력 (일부 테스트 규칙에서 실패로 처리됨)
- 카드 소유자/인증 관련 필드가 있다면 의도적으로 비정상 입력

## 확인 방법

- 결제창 실패 시 `/app/billing?result=fail`로 이동하는지 확인
- `/api/payments/toss/billing/issue` 호출이 발생하지 않는지 확인
